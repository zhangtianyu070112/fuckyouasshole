# 实习日记 — 航空驾驶舱模拟控制系统开发

## 7月4日

今天算是项目进入集中开发的第一天。之前已经把模拟数据发生器、GPWS音频告警、A*航路寻路和FMC交互面板四个基础模块搭好了，今天的工作集中在基础架构和仪表显示上。

上午先搞了字体系统。之前用的是SDL自带的小十字占位符，可读性太差，这次用B612和B612Mono两套TTF字体，做了纹理缓存（1024条上限，满了就清），新增了左对齐和居中对齐的渲染函数，字体文件放在resources/fonts/下面。然后调整了驾驶舱布局，从原来的2×2四仪表改成3×2六仪表——上面一排CAPT PFD、EICAS、FO PFD，下面一排CAPT ND、FMC、FO ND，分辨率适配2560×1600全屏桌面模式。

下午搭导航数据库。先硬编码了10个中国机场加18个航路点，后面肯定要扩展。哈希表接入了RouteGraph做O(1)节点查找，AVL树接入了FMC的LEGS页做字母序/航路序排序切换。A*寻路用了二叉最小堆和Haversine启发式，加了高度过滤，基本上从任意两个已知机场之间都能跑出航路来。

ND航路渲染也做了第一版——geo_to_screen经纬度转罗经花像素坐标，航路点用青色菱形、机场用品红方块、航线用绿色连线、活跃航段用品红高亮。FMC CDU按钮面板搭了框架：功能键×7、数字3×4、字母4×6、编辑键，便签本左对齐加闪烁光标。EICAS的CAS方形告警灯面板（50×28px, 4列）实时驱动12个告警。PFD的动态刻度带也做了基础参数——空速带px_per_kt=5.0、高度带px_per_ft=0.6、航向带px_per_deg=5.0。

## 7月6日

今天集中搞PFD和EICAS的显示重设计。

PFD的问题在刻度带上——空速/高度/航向三个磁带用int截断值计算刻度位置，导致平滑值250.3→250.9期间刻度纹丝不动。根因是int ias截断了小数部分，所有刻度位置用固定偏移量。改成float ias，刻度位置用(tick_value - current_float) * px_per_unit逐像素滚动，航向带同理从固定偏移改成连续差值(h - hdg) * px_deg。修完之后磁带终于顺滑了。

然后给PFD新增了VSI垂直速度指示器——从16px细条改成7%宽度完整垂直刻度条，±6000fpm量程，每1000fpm刻度线加偶数标注，青色三角指针加绿色数字读数随VS连续移动。加了smooth_vs平滑变量防止跳变。N1推力指示器放在FMA下方、姿态指示器上方，双发水平条形图（灰底+绿色/琥珀填充+红色100%参考线+数字%），加了smooth_n1[2]平滑。FMA重设计成4列布局——A/THR|ROLL|PITCH|AP，激活时绿/品红文字加暗色背景框，未激活灰色。航向带加了5°小刻度+10°大刻度+30°标注，N/E/S/W用TTF大字加方向角小字。

下午修了一个坑：FMC鼠标点击错位。渲染时screen_h用的是52%，点击检测用的是58%，6%的高度差导致点击绘制的按键实际命中上一行。定义了5个共享布局常量（SCREEN_MARGIN、TITLE_BAR_H、SCREEN_H_PCT、KB_GAP、KB_BOTTOM_GAP），渲染+LSK点击+CDU按键点击三个函数统一引用，消除硬编码。

## 7月8日

今天开始对接X-Plane 12，问题比预想的多。

第一个问题：UDP数据收不到。排查下来四个坑。一是udp_thread_worker把RPOS/DSEL订阅请求发到了自己的接收端口49005而不是XP的命令端口49000，修正为UDPThreadArgs拆成recv_port和xplane_cmd_port两个字段。二是DSEL订阅被跳过——xplane_subscribe里RPOS发送成功后return 0提前返回，27个数据组的DSEL从未发送，改成无条件发送RPOS+DSEL。三是缓冲区太小——XP12的捆绑式数据包6233字节远超原2048缓冲（WSAEMSGSIZE 10040），改成65536字节。四是XP12用"DATA*"（0x2A字节）而不是XP11的"DATA\0"做包标识，匹配逻辑要放宽。

数据能收到了但格式又不对。XP12把所有数据组合并到一个UDP包里（DATA* + N×36字节），旧代码只解析第一组，改成循环解析全部组。最头疼的是数据组索引大重排——XP12的160+数据组用了全新编号，比如发动机N1从26变41、N2从27变42、EGT从34变47、FF从37变45，自动驾驶从63拆成108（开关）+118（数值），NAV/COM从111拆成97+99+102。写了完整的映射表config/xplane12_data_groups.json。

单位制也有变化。XP12用英制（高度ft、VVI ft/min、燃油流量lbs/hr），之前所有×3.28、×196.85、×2.20462的转换都要移除。N1/N2发送0.0-1.0比率，自动检测≤2.0的值×100转百分比。磁航向用-999.0表示磁差不可用，回退到真航向。滚转要取反。

下午修了PFD姿态仪两个bug。天地填充的screen_horizon_y跟X坐标无关（恒为0），改成基于地平线两端点的逐行交叉积判断——逐行测试左右两端点是否同侧，同侧整行单色，异侧从交点分割。然后颜色宏又踩了坑：set_color(r, cond ? COL_SKY_BLUE : COL_GROUND_BROWN)，多值宏在?:里被逗号运算符截断，颜色值完全错乱，4处三元颜色调用全部改if/else分支。

## 7月12日

今天做Route A的完整C层实现——25条RREF订阅。

XP12的RREF协议是413字节固定请求包（RREF\0 + freq + request_id + path 400B null-padded），发送订阅后XP12回12字节响应（4B "RREF" + 4B request_id + 4B float value）。订阅了21个告警灯状态（bank_angle/stall_warning/gear_warning/GPWS/overspeed/windshear/autopilot_disconnect/engine_fire/fire_warning @4Hz, door/generator/anti_ice/fuel_quantity/oil_pressure/oil_temperature/voltage/ice @2Hz, hydraulic_pressure/quantity/cabin_altitude/pressurization @4Hz）+4个液压数值（hydraulic_pressure_psi[0][1] @4Hz, hydraulic_fluid_ratio[0][1] @2Hz）。xplane_parse_rref根据request_id→field映射表自动填入FlightDataValues的dref_*字段（21个bool+4个float）。

## 7月16日

今天主要攻克离港数据库和FMC DEP/ARR页面。

新建了departure_db模块（departure_db.c/h）。设计了DepartureAirport结构体，包含ICAO码、跑道列表（跑道号+长度+标高）、SID列表（程序名+对应跑道+航路点序列）。数据库容量256个机场，每个机场最多24条跑道和16个SID。数据从fmc_data.txt解析，自定义的文本格式——空行分隔机场，每行key=value。写了解析器处理跑道的多值字段（跑道号/长度/标高用逗号分隔）和SID的嵌套结构（SID名、适用跑道、航路点序列）。

FMC DEP/ARR页面用了一个三态状态机重写。state 0是概览页——显示DEP/ARR机场和SID/STAR/跑道信息，LSK L1输入ICAO码自动去数据库查，找到了就进入state 1，找不到提示"NOT IN DATABASE"。state 1是跑道选择——最多显示4条跑道，支持滚动翻页，选中后进入state 2。state 2是SID选择——按选中的跑道过滤可用SID列表，同样4条一组可滚动。选中SID后自动填入FlightPlan的sid字段，在LEGS页航路前插入SID的航路点序列。整个流程有清晰的消息提示（"AIRPORT OK, SEL RWY"→"RWY SET, SEL SID"→"SID SET"），R6始终是BACK键回退，LSK L6始终是返回。滚动偏移dep_scroll根据列表长度自动修正不会滚过头。

这个状态机设计比之前用scratchpad手动输文本合理太多了。之前要手动敲SID名称，很容易打错，而且根本记不住哪个跑道对应哪些SID。现在从数据库里选，又快又准。

FlightPlan结构体也扩展了几个新字段——transition_altitude_ft（过渡高度，默认18000ft）、descent_angle_deg（下降角，默认3.0°）、engine_out_accel_alt_ft（单发加速高度）。CLB和DES页面现在能正确显示TRANS ALT和DES ANGLE了，不再是空行。

LEGS页的导航逻辑也修了。PG DN之前是+10然后回绕到0，现在改成+5步进、上限卡在waypoint_count-5不绕回，更符合真实FMC的行为。航路点删除（LSK R行）后scroll自动修正不会出现空白页。之前遗留的9个测试文件（test.c、test_fmc_coords.c、test_fmc_render.py、test_parse.c等）也一并清理了。
