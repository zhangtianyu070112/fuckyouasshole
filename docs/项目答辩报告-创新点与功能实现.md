## 二、核心创新点（答辩重点）

### 🏆 创新点 1：双座驾驶舱 + 8 仪表完整布局

**原方案**：2×2 四仪表（PFD/ND/EICAS/FMC 各一）。

**实现**：**3×3 网格双座布局**，完整复刻商用客机机长/副驾驶双侧仪表：

```
┌──────────┬──────────┬──────────┐
│ CAPT PFD │  EICAS   │  FO PFD  │
├──────────┼──────────┼──────────┤
│ CAPT ND  │ LEFT FMC │ EICAS2   │
├──────────┼──────────┼──────────┤
│          │ RIGHT FMC│  FO ND   │
└──────────┴──────────┴──────────┘
```

- 分辨率自适应（最高 2560×1600 全屏）
- 每块仪表独立 `Instrument` 接口，`on_init/on_update/on_render/on_event/on_destroy` 统一生命周期
- 双 FMC 独立运行（机长/副驾驶各自输入航路）

### 🏆 创新点 2：空间哈希网格 — 23.5 万航路点实时查询

**原方案**：仅提及"哈希表、链表、AVL 树"作为通用数据结构。

**实现**：自研 **Grid-based Spatial Hash**（`spatial_hash.c`）：
- 1° 纬度带 × 动态经度宽度（`1.0/cos(lat)` ≈ 每格 ~60 NM）
- 64-bit 打包键：`(lat_band << 32) | lon_sector`
- **航向优先 5×5 邻域搜索**：按 cosine 相似度排序邻域网格，优先搜索航向前方
- 2,003 桶哈希表，~235k 条目，全部堆分配
- 加载真实导航数据：`earth_fix.dat`（~198k 航路点）+ `earth_nav.dat`（~37k VOR/NDB/机场）

**技术亮点**：并非简单使用现成的哈希表——而是针对导航数据的地理空间特性，从头设计了带航向优先的网格空间索引，查询效率远超线性扫描。

### 🏆 创新点 3：A* 航路寻路引擎 + 链式前向星图

**原方案**：FMC "航路编辑、航点管理"。

**实现**：完整的 **图论航路规划系统**（`route_graph.c`）：
- **链式前向星**（Chain Forward-Star）邻接表：O(1) 边遍历
- **A\* 最短路径**：Haversine 大圆距离启发（可纳，保证最优解）
- 二叉最小堆开放集 + 世代标记闭合集
- 航路高度层约束过滤
- 机场→航路网自动连接（SID/STAR 门户，200 NM fallback）
- **对比 Dijkstra：1,000 节点北京→上海约 7.5× 加速**

### 🏆 创新点 4：X-Plane 12 双向原生 UDP 通信

**原方案**：仅"从 X-Plane 11 接收实时飞行参数"。

**实现**：**全双工原生 UDP**，不依赖任何第三方插件：

| 方向 | 协议 | 功能 |
|---|---|---|
| **XP12 → Cockpit** | RPOS + DSEL + DATA | 27 数据组订阅，B738 完整飞行参数 |
| **Cockpit → XP12** | DREF（509B 固定包） | 写入巡航高度/速度/首段航向到 AP |
| **Cockpit → XP12** | CMND | 完整 CDU 键盘镜像（`sim/FMS/key_*`） |

**技术难点攻克**（`progress-2026-07-05`）：
- XP12 数据组索引全部重排（160+ 组全新编号），从零逐组测绘
- DATA 包第 5 字节：XP12 用 `0x2A`（`"DATA*"`）而非 XP11 的 `0x00`
- DREF 包格式：XP12 509 字节固定包 vs XP11 42-44 字节可变长
- 多组合并解析：单一 UDP 包含 N×36 字节多数据组

### 🏆 创新点 5：39 种 GPWS + 系统告警 + 实时音频合成

**原方案**：10 种 GPWS 告警。

**实现**：**三级告警体系**（`alert_system.c`）：

| 层级 | 数量 | 内容 |
|---|---|---|
| **GPWS 近地告警** | 13 种 | PULL UP, WINDSHEAR, TERRAIN, SINK RATE, TOO LOW GEAR, TOO LOW FLAPS, GLIDESLOPE, BANK ANGLE, OVERSPEED, MINIMUMS 等 |
| **系统参数告警** | 10 种 | 发动机超限、液压故障、电气异常、防冰等 |
| **C 层 RREF 告警** | 25 条 DREF | 直接读取 XP12 内部告警标志位 |
| **高度自动喊话** | 10 档 | 500/400/300/200/100/50/40/30/20/10 ft |
| **测试提示音** | 39 种 | 每种告警配备独立测试触发 |

- SDL2 音频回调实时合成波形（方波/正弦波/锯齿波），**零外部依赖**
- 优先级抢占 + 冷却计时器防重复播报
- 架构预留 WAV 真人语音替换接口
- 无音频设备时优雅降级

### 🏆 创新点 6：AI 副驾驶系统（LoRA 微调 ChatGLM3-6B）

**原方案**：无 AI 相关规划。

**实现**：完整的 **AI 驾驶舱态势感知** 管线：

```
C Cockpit → WebSocket → FastAPI 推理服务器 (ChatGLM3-6B QLoRA)
         ← token stream ← AI 副驾驶建议 → EICAS 显示
```

| 环节 | 实现 |
|---|---|
| **训练数据** | 37,700 条 ShareGPT 格式，80 飞行剖面 × 39 告警类型 |
| **基座模型** | ChatGLM3-6B，QLoRA 4-bit（r=16, α=32） |
| **推理部署** | llama.cpp GGUF Q4_K_M，~4GB，全 GPU offload |
| **通信协议** | WebSocket RFC 6455，非阻塞 C 客户端（`ai_advisor.c`，937 行） |
| **延迟目标** | 首 token <60ms，单 token <35ms |
| **安全设计** | GPWS 硬告警不可绕过，LLM 仅为补充软建议 |
| **容错** | 断连指数退避重连（2s→30s），服务器不可用时优雅降级 |

**技术创新**：
- 从零手写 WebSocket RFC 6455 客户端（C 语言，仅依赖 Winsock2，无第三方库）
- 事件驱动状态推送（参数变化超阈值才发送，≤200ms 间隔）
- 流式 token 逐字接收 + 帧重组

### 🏆 创新点 7：ND 四模式导航显示器

**原方案**：单一 MAP 罗盘地图模式。

**实现**：四种完整导航模式（`nd.c`，1,264 行）：

| 模式 | 功能 | 快捷键 |
|---|---|---|
| **MAP** | 罗盘地图 + 空间哈希实时导航叠加（默认） | — |
| **VOR** | 全罗盘 + VOR 航道指针 + CDI 偏差 + DME 距离 | `m` |
| **APP** | ILS 进近 — LOC 偏差 + GS 偏差 + 航向刻度带 | `m` |
| **PLN** | 正北朝上静态计划视图，显示完整航路 | `m` |

- 航路点（青色菱形）、机场（品红方块）、航线（绿色连线）、活跃航段（品红高亮）
- `w`/`a`/`r`/`v`/`c` 快捷键独立控制各类导航元素显隐
- `+`/`-` 量程缩放
- 顶部信息栏：TRK|MAG 地面航迹角 + 目标航路 ETA + 风速箭头
- 侧面板：COM1/COM2/NAV1/NAV2/DME/XPDR/OAT

### 🏆 创新点 8：PFD 全浮动连续滚动刻度 + VSI/推力指示器

**原方案**：基本姿态/速度/高度/航向显示。

**实现**：**高精度机械式连续滚动**（`pfd.c`，1,458 行）：

- `float` 连续差值驱动刻度位置（非 `int` 截断），逐像素平滑滚动
- 新增 **VSI 垂直速度指示器**：±6,000 fpm，青色三角指针
- 新增 **N1 双发推力指示器**：水平条形图 + 红色 100% 参考线
- 新增 **FMA 飞行模式通告**：4 列布局（A/THR | ROLL | PITCH | AP）
- 马赫数、无线电高度、AP 航向/高度/速度预选 bug
- 姿态仪天地染色：交叉积算法逐行判定 + 地平线数学重合
- 航向 0°/360° 平滑跨越（`exp_smooth_angle()`）

### 🏆 创新点 9：EICAS 发动机圆弧表盘 + 全数据面板

**原方案**：基本发动机参数和告警。

**实现**（`eicas.c` 668 行 + `eicas2.c` 351 行）：

- **N1/EGT 180° 半圆弧表盘**（左右各一），灰色轨道 + 白色刻度/指针 + 超标红区
- CAS 方形告警灯面板（4 列 × 实时驱动）
- 告警事件日志（链表存储，最近 3 条滚动显示）
- EICAS2 副屏：辅助发动机参数/液压/电气/增压/防冰/刹车温度/APU 完整数据面板
- 燃油量框（带边框，LBS×1000，大字体）
- TAT 总温（含 Ram Rise 修正 `(TAS/100)²×0.55`）

### 🏆 创新点 10：FMC CDU 完整交互 + 手动航路点编辑

**原方案**："航路编辑、航点管理"。

**实现**（`fmc.c`，2,232 行，项目最大单文件）：

| 页面 | 功能 |
|---|---|
| **IDENT** (1/10) | 飞机/发动机型号、导航数据库统计 |
| **RTE** (2/10) | ORIGIN/DEST 输入 → ACTIVATE 触发 A\* 自动规划 |
| **LEGS** (3/10) | 航点列表（航向 + 累计距离），**手动插入/复制/删除航路点** 🆕 |
| **PERF** (4/10) | 巡航高度/速度/燃油/成本指数 |
| **PROG** (5/10) | 距目的地距离、ETA、地速、高度、偏航距 |
| **RADIO** (6/10) | COM1/2、NAV1/2、XPDR、DME 频率 |
| **DEP/ARR** (7/10) | SID/STAR 程序 + 跑道配置 |
| **CLB** (8/10) | 爬升目标速度/马赫数/速度限制 |
| **CRZ** (9/10) | 巡航高度/目标马赫数/最优/最大高度层 |
| **DES** (10/10) | 下降终点高度/目标速度/速度限制 |

**手动航路点编辑**（2026-07-10 新增）：
- LEGS 页 LSK L1-L5：输入 ident → **插入前方** / 空 → **复制到便签本**
- LEGS 页 LSK R1-R5：输入 ident → **插入后方** / 空 → **删除该行**
- O(1) ident 查找：50 万桶哈希表索引 23.5 万航路点
- 底层 `flight_plan_add_waypoint` / `flight_plan_remove_waypoint` 完整实现

### 🏆 创新点 11：客舱航线显示（高德地图 + HTTP 服务）

**原方案**："通过 HTTP 接入高德地图 API，实现航班位置、航迹、天气等辅助信息展示"。

**实现**：

  - 飞机三角图标（跟随航向旋转，ease-out 位置插值）

  - 起降机场标记 + 航点标签
  - 底部数据栏：GS/ALT/HDG/OAT/距目的地/ETA + 航程进度条

## 三、技术架构深度

### 3.1 多线程并发架构

```
┌─ 渲染主线程 ─────────────────────────────────────┐
│  app_run(): 60 FPS 主循环                          │
│  ├─ SDL_PollEvent() → 事件分发 → 仪表 on_event     │
│  ├─ flight_data_snapshot() → mutex 快照（极短临界区）│
│  ├─ 8× Instrument on_update() + on_render()        │
│  ├─ ai_advisor_update() → WebSocket 非阻塞推送      │
│  └─ SDL_RenderPresent()                            │
├─ 数据线程 ────────────────────────────────────────┤
│  UDP recv (49005) → RPOS/DSEL/DATA 解析             │
│  flight_data_update() → mutex 写入                  │
│  或 mock_data_thread() → 每 50ms 写入模拟数据       │
└────────────────────────────────────────────────────┘
```

- 渲染/网络/逻辑线程分离
- FlightData mutex 快照机制（读锁仅 `memcpy`，极短临界区）
- 非阻塞 AI WebSocket 客户端（主渲染循环 60Hz 安全调用）

## 六、项目亮点总结（答辩关键词）

1. **深度定制数据结构** — 6 种数据结构从零实现，空间哈希网格针对导航场景专门设计

9. **多线程并发** — 渲染/网络/AI 三线分离 + mutex 快照低延迟同步
