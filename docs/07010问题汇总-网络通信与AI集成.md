## 一、网络通信问题

### 问题 1: 端口占用 — 本地 8090 被 Windows 预留

**现象**: SSH 隧道 `-L 8090:localhost:8090` 报 `bind [127.0.0.1]:8090: Permission denied`

**原因**: 多个进程（PID 9492 等）已占用 127.0.0.1:8090 和 ::1:8090，可能是 Hyper-V / Windows 容器预留。

**解决**: 换用高位端口 58090，避开冲突区。

**相关配置**: `config/default.cfg` → `inference_port = 58090`

---

### 问题 2: 配置文件端口重复

**现象**: 改端口后 cockpit.exe 连接仍然失败。

**原因**: `default.cfg` 中 `[ai]` 节有两条 `inference_port` 行（58090 和 8090），配置解析器读到后者覆盖了前者。

**解决**: 删除重复行，只保留 `inference_port = 58090`。

**教训**: 用 `grep` 检查配置不要有重复 key。

---

### 问题 3: WebSocket 帧解析 — `{"done": true}` 空格不匹配

**现象**: 推理服务器正常返回 token，但 cockpit 控制台显示 `DONE received, pending_len=0`，token 从未被累积。

**原因**: Python `json.dumps({"done": True})` 输出 `{"done": true}`（冒号后有空格），C 端用 `memcmp(payload, "{\"done\":true}", 13)` 精确匹配无空格版，永远失败。

**解决**: 改用子串搜索：`strstr(payload, "\"done\"") && strstr(payload, "true")`。

**同类型**: token 帧 `{"token": "..."}`（有空格）也被 `memcmp` 精确匹配 `"token":"`（无空格）漏掉。最终改为找最后一对引号提取 value。

**教训**: **永远不要用精确字符串匹配 JSON**。JSON 允许任意空白字符。用子串搜索或 JSON parser。

---

### 问题 4: 推送频率过高导致推理永远无法完成

**现象**: EICAS 底部一直显示 `-- waiting --`，GPU 占用 60% 但从未显示建议文本。

**原因**: 
- 飞行数据每 100ms 更新一次（10Hz mock 数据 / 20Hz X-Plane）
- C 端每 200ms 检测到状态变化 → 推送新帧 → 调用 `do_push()` 清空 `pending[]`
- 服务器推理需要 2-10 秒
- 推理完成前 `pending` 已被下一次推送清空 → 永远无法积累到 `done`

**解决**: 加 `inferring` 门控——推理中禁止推送新状态（离散紧急事件除外，改为排队等 `done` 后再推）。

**代码位置**: `src/ai/ai_advisor.c` — `AIAdvisor.inferring` + `has_pending_push`

---

### 问题 5: 多告警并发时离散事件反复打断推理

**现象**: 同时触发 BANK ANGLE + STALL 时 ADVISORY 卡在 waiting。

**原因**: 离散告警（gear/ap/warning/caution）变化绕过了 `inferring` 门控，每个事件都立即推送 → 服务器不断取消旧推理。

**解决**: 推理中离散事件不再立即推送，改为设置 `has_pending_push` 标志，等 `done` 后再推最新状态。

**代码位置**: `src/ai/ai_advisor.c` — push 逻辑区 `if (ai->inferring)` 分支

---

### 问题 6: WebSocket Pong 帧未掩码导致服务器断开连接

**现象**: 首次推理成功，之后每隔约 10 秒连接断开重连（`FRAME op=8` close code 1002 "incorrect masking"），后续推理全部失败。

**原因**: RFC 6455 §5.5 规定 **客户端→服务器的所有帧（含控制帧）必须掩码**。Ping/Pong 回复帧缺少 MASK 位 + 4 字节掩码 key。服务器收到未掩码 Pong 后拒绝后续数据帧。

**解决**: Pong 帧构建加上 `0x80 | len`（MASK 位）+ 4 字节随机掩码 + XOR payload。

**代码位置**: `src/ai/ai_advisor.c` — `case 0x9` Ping→Pong 分支

---

### 问题 7: 掩码生成器随机性不足

**现象**: 配合问题 6 一起排查时发现。

**原因**: `ws_gen_mask_key()` 仅用 `SDL_GetTicks()` 经简单位运算生成 4 字节，短时间内连续调用可能重复。且可能产生全零字节（服务器可能拒绝）。

**解决**: 
- 引入 SplitMix32 哈希 + 栈地址 ASLR 熵 + 递增计数器
- 每字节 `| 1` 确保非零

**代码位置**: `src/ai/ai_advisor.c` — `ws_gen_mask_key()`

## 二、渲染问题

### 问题 8: EICAS 底部 AI 区域看不到

**现象**: 推理正常但 EICAS 上无 ADVISORY 显示。

**原因**: 
1. 油量盒（Fuel QTY）动态高度覆盖到 `rect->h - 8`，遮挡了 AI 区域（`rect->h - 16`）
2. 字体太小——B612 0.3~0.45 缩放在缩小后的纹理上不可见

**解决**: 
- 油量盒改为固定 60px 高度
- AI 区域上移并加高到 52px，加分隔线
- 字体放大到 0.70~0.85

**代码位置**: `instruments/EICAS/eicas.c` — `eicas_on_render()`

---

### 问题 9: 中文汉字全部显示为方框 □

**现象**: ADVISORY 文本英文正常，中文全是 □。

**原因**: B612 字体家族是纯拉丁字体，不含 CJK 字形。

**解决**: 新增 `FONT_CJK` 字体 ID，映射到阿里巴巴普惠体（`ALIBABAPUHUITI-2-45-LIGHT.TTF`），仅 AI 文字行使用 CJK 字体，其余全部保持 B612。

**涉及文件**:
- `src/utils/font_manager.h` — 新增 `FONT_CJK` 枚举值
- `src/utils/font_manager.c` — 新增字体文件名映射
- `resources/fonts/` — 复制普惠体 ttf
- `instruments/EICAS/eicas.c` — AI 文字行用 `FONT_CJK`

---

## 三、告警优先级问题

### 问题 10: 失速/大坡度等致命告警被除冰等次要告警淹没

**现象**: 飞机进入失速尾旋（IAS < 60kt, roll 80°, VS -20000fpm）时 AI 仅报告 "除冰电门未打开"。

**原因**: 
1. C 端只发了 `master_warning=1`，未告知具体告警类型
2. 工作站 prompt 构造未将告警字段放在优先位置

**解决（C 端）**:
- JSON 新增 7 个告警位：`stall`, `gpws`, `windshear`, `overspeed`, `bank_angle`, `gear_warn`, `engine_fire`
- 不依赖 DREF——从原始飞行参数**实时计算**告警（`compute_alerts()`）

**解决（工作站端，参考 `docs/工作站TODO-告警优先级修复.md`）**:
- `build_prompt()` 开头注入告警块，致命告警（火警/失速/GPWS/风切变）用 `‼【致命告警】` 标记放在 prompt 最前面
- System Prompt 加入优先级规则
