# 工作站推理服务器 — 需求规格

> **文档版本**: 1.0  
> **日期**: 2026-07-10  
> **用途**: 在 Linux 工作站（2× RTX 3090, 24GB×2）上重建 ChatGLM3-6B QLoRA AI 副驾驶推理服务  
> **对应客户端**: `src/ai/ai_advisor.c`（Windows 开发机，同局域网 WebSocket 连接）

---

## 一、硬件与模型

| 项目 | 规格 |
|------|------|
| GPU | 2× NVIDIA RTX 3090 (24GB VRAM each) |
| 基座模型 | ChatGLM3-6B (`THUDM/chatglm3-6b`) |
| 微调方法 | QLoRA 4-bit (r=16, alpha=32) |
| 训练框架 | LLaMA-Factory + DeepSpeed ZeRO-2 |
| 导出格式 | GGUF Q4_K_M |
| 模型文件 | `models/cockpit-advisor-q4km.gguf`（约 4GB） |
| 推理引擎 | llama.cpp (via `llama-cpp-python`) |

---

## 二、推理性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **首 token 延迟** | < 60ms | 从收到 WebSocket 消息到第一个 token 推送 |
| **单 token 生成速度** | < 35ms/token | Q4_K_M 在 3090 上的典型值 |
| **完整响应延迟** | < 250ms | 一次推理（max 128 tokens）的总时间 |
| **模型加载时间** | < 10s | 服务器启动时一次性加载 |
| **GPU VRAM 占用** | < 6GB | Q4_K_M 6B 模型全 GPU offload |
| **吞吐量** | ≥ 2 req/s | 单连接场景（驾驶舱只有一个客户端） |

### 延迟预算分解

```
收到 WebSocket 帧 → JSON 解析         < 1ms
JSON → 构造 prompt 文本              < 1ms
llama.cpp tokenize                   < 5ms
首个 token 生成 (prefill)             ~30ms
每个后续 token 生成 (decode)          ~25ms
token → JSON → WebSocket 发送        < 1ms
─────────────────────────────────────────
首 token 端到端:  ~40ms  ← 满足 <60ms
128 tokens 完整:   ~40 + 127×25 ≈ 3.2s  ← ⚠ 需优化
```

> ⚠ **注意**: 128 tokens 全量生成约需 3 秒。建议 `max_tokens` 设为 **64** 以控制在 1.6 秒内，或提前截断（输出目标 ≤50 中文字符）。

## 三、网络与通信协议

### 3.1 连接规格

| 项目 | 值 |
|------|-----|
| 协议 | WebSocket (RFC 6455) |
| 端口 | **8090** |
| 绑定地址 | `0.0.0.0`（监听所有网卡） |
| 连接数 | 1（单客户端，驾驶舱模拟器） |
| 连接模式 | **持久连接**（不断开，复用） |
| 心跳 | WebSocket Ping/Pong，30s 间隔 |
| 传输编码 | JSON 文本帧（UTF-8） |

### 3.2 客户端 → 服务器（飞行状态推送）

**频率**: 事件驱动，状态变化超过阈值时推送，上限 200ms 间隔（5 Hz）

**JSON 格式**（约 350 字节）：

```json
{
  "alt_ft": 35000,
  "agl_ft": 34000,
  "ias_kts": 280,
  "vs_fpm": 50,
  "mach": 0.785,
  "roll_deg": 2.1,
  "pitch_deg": 2.5,
  "heading": 125,
  "n1_pct": [60.2, 60.8],
  "egt_c": [700, 705],
  "flap_ratio": 0.0,
  "gear_down": 1,
  "fuel_lbs": 8500,
  "oat_c": -39,
  "ap_engaged": 1,
  "master_warning": 0,
  "master_caution": 0
}
```

**C 端推送阈值**（只有变化超过阈值才推送）：

| 参数 | 阈值 |
|------|------|
| 高度 (alt_ft) | ±50 ft |
| 空速 (ias_kts) | ±5 kts |
| 垂直速度 (vs_fpm) | ±100 fpm |
| 航向 (heading) | ±3° |
| 横滚 (roll) | ±2° |
| 俯仰 (pitch) | ±1° |
| N1 | ±2% |
| EGT | ±10°C |
| 襟翼 | ±5% |
| 燃油 | ±200 lbs |
| 起落架/AP/告警灯 | 任意变化（离散量） |

### 3.3 服务器 → 客户端（Token 流式推送）

**格式**: 每个 token 一个独立的 WebSocket 文本帧

```
{"token": "注"}
{"token": "意"}
{"token": "："}
{"token": "下"}
{"token": "降"}
{"token": "率"}
...
{"done": true}
```

**结束标记**: 每次推理完成后发送 `{"done": true}`。

**错误处理**: 推理失败时发送 `{"error": "inference_failed"}`。

### 3.4 WebSocket 路径

```
ws://<工作站IP>:8090/api/copilot/stream
```

C 端握手请求：
```
GET /api/copilot/stream HTTP/1.1
Host: <ip>:8090
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Key: <base64 16 random bytes>
Sec-WebSocket-Version: 13
```

## 四、Prompt 构造规范

> **注意**: Prompt 构造（飞行阶段推断 + 中文格式化）在 **C 端** (`src/ai/ai_advisor.c` 的 `build_user_prompt()`) 完成。
> 工作站 Python 服务器只接收**原始中文 prompt 文本**，直接喂给 llama.cpp，不含任何航空领域逻辑。
> 这使工作站成为可复用的通用 LLM 推理服务。

### 4.1 System Prompt（Python 端唯一常量）

```
你是B737-800驾驶舱AI副驾驶。监控飞行参数，用中文提供简洁的操作建议或态势说明。输出不超过50字。一切正常时报告"正常"，异常时指出具体问题和建议操作。你提供的建议仅供参考，不可替代SOP和GPWS硬告警。
```

### 4.2 C 端构建的 User Prompt 示例

| 参数 | 值 | 说明 |
|------|-----|------|
| max_tokens | 64（建议）| 目标输出 ≤50 中文字符 |
| temperature | 0.1 | 低温度保证确定性输出 |
| top_p | 0.9 | 适度 nucleus sampling |
| stream | true | 必须开启流式输出 |
| repeat_penalty | 1.1 | 防止重复输出 |

## 六、C 端预期行为

服务器应了解客户端的以下行为模式，以便调试：

1. **启动** — 客户端启动后立即尝试 TCP 非阻塞连接，完成后发送 WebSocket 握手
2. **首次连接** — 握手成功后立即推送当前飞行状态 JSON
3. **持续运行** — 飞行状态变化时按 ≤200ms 间隔推送，服务器应流式返回 token
4. **断开** — 连接断开后客户端 2s 后开始重连，指数退避（4s → 8s → ... → 30s 上限）
5. **优雅降级** — 服务器不可用时 cockpit.exe 正常运行，EICAS 底部不显示 AI 行

- [ ] 如防火墙不可配：从开发机 `ssh -L 8090:localhost:8090 -N -f workstation`

### WebSocket 连通性测试脚本

```python
# test_ws.py — 从开发机运行
import asyncio, json, time, websockets

WORKSTATION = "192.168.1.100"  # 改为实际 IP

async def test():
    t0 = time.time()
    async with websockets.connect(f"ws://{WORKSTATION}:8090") as ws:
        print(f"Connected in {(time.time()-t0)*1000:.0f}ms")
        await ws.send(json.dumps({
            "alt_ft": 35000, "agl_ft": 34000, "ias_kts": 280,
            "vs_fpm": 50, "mach": 0.78, "roll_deg": 2.1,
            "pitch_deg": 2.5, "heading": 125,
            "n1_pct": [60.2, 60.8], "egt_c": [700, 705],
            "flap_ratio": 0.0, "gear_down": 1,
            "fuel_lbs": 8500, "oat_c": -39,
            "ap_engaged": 1, "master_warning": 0, "master_caution": 0
        }))
        t_first = None
        tokens = []
        async for msg in ws:
            data = json.loads(msg)
            if t_first is None:
                t_first = time.time()
                print(f"First token in {(t_first-t0)*1000:.0f}ms")
            if "token" in data:
                tokens.append(data["token"])
                print(data["token"], end="", flush=True)
            if "done" in data:
                break
        print(f"\nTotal: {len(tokens)} tokens in {(time.time()-t0)*1000:.0f}ms")
        print(f"Full text: {''.join(tokens)}")

asyncio.run(test())
```
